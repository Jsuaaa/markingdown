# Markingdown

A WYSIWYG Markdown editor built for developers. Designed to visualize and edit plans generated by Claude Code, with a clean editing experience.

Built with Electron, React, and TipTap.

Markingdown — Editor view

## Features

### Editor

- **True WYSIWYG** — Write markdown and see it rendered in real time. No split panes, no preview toggle.
- **Slash commands** — Type `/` to insert headings, code blocks, tables, lists, task lists, blockquotes, dividers, and formatting marks.
- **Syntax highlighting** — Code blocks with language detection via Lowlight (150+ languages).
- **Tables** — Full GFM table support with header rows.
- **Task lists** — Interactive checkboxes that persist state.
- **TODO annotations** — Special `[TODO]:` blockquotes for marking action items.



### Claude Code Integration

- **ANSI cleanup on paste** — Terminal output with ANSI escape codes is automatically stripped and normalized.
- **Plan detection** — Pasted Claude Code plans are auto-converted to clean markdown.
- **Drag & drop** — Drop `.md`, `.markdown`, or `.txt` files directly into the editor.

### File Management

- **Multi-tab plans** — Work on multiple documents simultaneously with a sidebar tab system.
- **Auto-save** — Documents with an existing file path are saved automatically (2s debounce).
- **Auto-title** — Titles are extracted from the first heading in the document.
- **Reorderable tabs** — Drag to reorder plans in the sidebar.
- **Persistent state** — All open plans are saved to localStorage and restored on launch.



### Export


| Format   | Description                                    |
| -------- | ---------------------------------------------- |
| Markdown | Save raw `.md` files                           |
| PDF      | Styled export via Electron's `printToPDF`      |
| HTML     | Self-contained HTML with embedded CSS          |
| LaTeX    | Export via Pandoc subprocess (requires Pandoc) |


### Theming

- **Light & dark mode** — Automatic via `prefers-color-scheme`, or toggle manually.
- **Built-in presets** — Warm Cream (light) and Tokyo Night (dark).
- **Full color customizer** — Adjust editor background, text, headings, sidebar, toolbar, accents, code blocks, and borders.
- **Custom themes** — Save and manage your own themes.





### Keyboard Shortcuts


| Shortcut             | Action                 |
| -------------------- | ---------------------- |
| `Cmd/Ctrl + N`       | New plan               |
| `Cmd/Ctrl + O`       | Open file              |
| `Cmd/Ctrl + S`       | Save                   |
| `Cmd/Ctrl + Shift+S` | Save as                |
| `Cmd/Ctrl + W`       | Close tab              |
| `Cmd/Ctrl + B`       | Toggle sidebar         |
| `Cmd/Ctrl + Shift+L` | Toggle theme           |
| `Cmd/Ctrl + Shift+T` | Theme customizer       |
| `Cmd/Ctrl + 1-9`     | Switch to tab N        |
| `Escape`             | Clear formatting marks |


## Tech Stack


| Layer        | Technology                                             |
| ------------ | ------------------------------------------------------ |
| Desktop      | Electron 41                                            |
| Frontend     | React 19 + TypeScript 5.9                              |
| Bundler      | Vite 8 + vite-plugin-electron                          |
| Editor       | TipTap v2 + tiptap-markdown                            |
| State        | Zustand (localStorage persistence)                     |
| Highlighting | Lowlight (highlight.js)                                |
| Export       | remark / rehype (HTML), Electron (PDF), Pandoc (LaTeX) |

## Getting Started

### Prerequisites

- Node.js 18+
- npm

### Install

```bash
git clone https://github.com/yourusername/markingdown.git
cd markingdown
npm install
```

### Development

```bash
npm run dev        # Start Vite dev server + Electron
```

### Build

```bash
npm run build      # TypeScript check + Vite production build
npm run dist       # Build + package with electron-builder (macOS .dmg)
```

### Lint

```bash
npm run lint       # ESLint
```
